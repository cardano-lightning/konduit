# task-003 research

- runtime-selection finding: the current `CardanoConnector` trait remains non-object-safe because it returns `impl Future`, so the smallest truthful runtime-selection shape for both `konduit-server` and `konduit-cli` is a small enum wrapper that delegates the trait rather than a trait object refactor
- server-readiness finding: `konduit-server/src/main.rs` already builds the Cardano backend and awaits `admin::Service::new(...).await?` before `server.run()`, so UTxO RPC reachability and live network match can stay in `cardano/args.rs` while live protocol-parameter derivation and reference-script resolution remain truthful startup blockers through the existing admin-service boot path
- utxorpc network finding: the `utxorpc` crate helper only exposes raw `read_genesis()` bytes, but the generated gRPC query client still returns parsed `read_genesis_response::Config::Cardano(genesis)` including `network_magic` and `network_id`; that is sufficient to derive the live Cardano network without falling back to static presets
- network-mapping finding: Konduit can truthfully map live Dolos network data through `cardano_sdk::Network::{MAINNET_MAGIC,PREPROD_MAGIC,PREVIEW_MAGIC}`; `network_id = testnet` is shared by preview and preprod, so `network_magic` is the authoritative discriminator for this runtime validation
- CLI truthfulness finding: explicit UTxO RPC network selection cannot reuse the old CLI `KONDUIT_NETWORK` default of `mainnet`; for `utxorpc`, missing network must remain a configuration error, while Blockfrost can still infer or default its display/setup network separately
- CLI drift finding: Blockfrost project-id inference must only run when `backend=blockfrost`; otherwise stale Blockfrost env state can silently rewrite UTxO RPC config output and generated addresses during `fill()`
- UX finding: `show address` commands do not need a live connector and should derive from config network directly so config display and address-generation remain usable when runtime-only backend fields are absent; tip and tx flows still need live connector validation
- verification finding: task-local coverage for backend selection and config truthfulness was necessary in task-003 itself, not only in `task-101`; final focused tests now cover missing Blockfrost project id, missing UTxO RPC URI, missing explicit UTxO RPC network, network mismatch rejection, and stale Blockfrost env values being ignored for UTxO RPC
